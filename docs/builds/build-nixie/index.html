Meta 🔗

Previously… 🔗

I saw this project, DekuNuken/exixe, and felt that hey, this is really cool and what I could use! But, I figured I’d like them differently.

For starters, I’d like them to run over I2C instead of SPI, to make it easier with strings of nixies. I’d also like to use firmware I’ve already developed (eg my contiki port for msp430) and chips I already had (msp430), aaaand I thought I could do a cleaner job with the layout of the transistors.

So I made my own thing.

This one 🔗

An MSP430G2553 acts as I2C slave. It receives commands and controls the transistors that switch the Nixie tube. Commands include simple ones such as “turn on number 4” or “turn off”, or “fade from 3 to 4”.

One transistor per Nixie digit controls the low-current but high-voltage output - the high-voltage-rated BF820W, which costs about 0.32€ in MOQ 1.

The board accepts I2C (SDA, SCL), 3.3V and GND power for the MSP430, and high voltage for the Nixie tubes.

Programming is done via a programming header.

Result 🔗

Hmmmm. The msp430 is immensely hard to program. Perhaps bad signal integrity on the programming pins, or bad reset pulse (high capacitance?). Need to investigate, some rainy day.

This much I know already though - for revision 2.0:

  • PCB: slightly longer pads on msp430 to make it easier to solder by hand
  • PCB: slightly larger drill holes on nixie legs (ie the 12-ish of them), it’s now a press fit
  • PCB: slightly larger solder pads for the headers
  • PCB: redesign programming header:
    • reuse VCC, GND from interface header
    • remove TXD, RXD - they are not used for programming
    • reorder to follow LP order
  • PCB: extend PCB edge upwards to fully cover the IN-12A, now it peeks out over the PCB edge
  • PCB: perhaps larger footprint for the 180V current limiting resistor? now 1206 but P = ca 0.6W, ie may need larger package
  • PCB: silkscreen, U1 unncessary
  • PCB: silkscreen, show pin one on msp430 better
  • PCB: silkscreen, description on headers hard to read
  • PCB: routing, liten knyck på TEST strax under msp430
  • PCB: i2c addr are at the wrong places, should be [0, 1, 2], is now [2, 0, 1] from bottom. easy fix, routing is easy
